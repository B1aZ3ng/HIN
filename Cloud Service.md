# SaaS
- Software as a service
- ready to use applications
- fully managed, ready to use software delvered over the internet, typically via subscription.
- User manages: nothing, just the application usage and data
- Provider manages everything
- Eg. Microsoft 365
- Least customisable, but no need to have any codig manages
- handles everything, user doesnt need to do anything to use
# PaaS
- Platform as a service
- Cloud computing model that provides devs with platform to build, delp and manage applications 
- Uside by software development
- allows teams of engineers to write, test, deploy, pdate and manage software in the clod
- eg. Google App Engine, ISAMS, Github
- takes away hardware, os management etc
- server which software is installed but you need to create the application yourself -  mostly customisable
- handles the basic software and hardware
# IaaS
- Infrastructure as a service
- Provided on demand access to IT resources like virtual servers, storage and networks over the internet
- AWS
- literally pre-set-up remote server
- handles basically anything hardware-wise
